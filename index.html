When Gradients Collide:

We extend TextGrad to optimize an LLM judge prompt across four evaluation criteria at once. Each step runs a 4-stage loop (task → loss → gradient → optimizer). By varying whether each stage processes tasks Separately or Combined, we get four decomposition modes (SSS, SSC, SCC, CCC). We find that combining criteria breaks the optimizer in two distinct ways.

Abstract

Customizing an LLM judge to a specific problem or domain often involves optimizing its prompt across multiple evaluation criteria simultaneously. Textual gradient methods automate this for a single judge criterion, however they produce natural-language critiques, not numerical vectors. Thus, the conflict-resolution toolkit of multi-task learning (PCGrad, MGDA) does not apply to this multi-objective textual gradient setting. We extend TextGrad to the multi-objective setting and test four decomposition modes of textual gradient optimizers by varying how much cross-objective information the loss, gradient and optimizer LLMs share. We find the gradient's task-focus drops by 59% (9.0 to 3.7 out of 10) when the gradient LLM must provide feedback on multiple criteria jointly. Separately, we observe that naively combining single-objective optimized instructions into a single prompt degrades Spearman ρ from 0.305 to 0.220 (−0.085). These results identify two separable failure modes: optimization-time gradient dilution and inference-time instruction interference, which together constrain the design space for multi-objective judge optimization using textual feedback.

Failure Mode 1: Gradient Dilution

The first failure happens during optimization. We measure each textual gradient for task-focus: how targeted its improvement suggestions are to a single criterion (scored 1–10 by an LLM evaluator). When the gradient LLM processes each task separately (modes Single, SSS, SSC), gradients are sharply focused, scoring a mean of 9.0. But when it must reconcile feedback from all four criteria in one call (modes SCC, CCC), task-focus collapses to 3.7 — a 59% drop with zero overlap between the two groups.

This is a structural cliff, not a model artifact: swapping the gradient LLM to a stronger model (DeepSeek-V4-Pro) reproduces the same SSC→SCC collapse. The optimizer still faithfully applies whatever gradient it receives (feedback adherence stays high at 7.7–8.8), which means the bottleneck is gradient quality, not optimizer compliance.

Optimization Stagnates

The downstream effect is stark. In 6 of 10 configurations, optimization never beats the initial generic prompt (“Rate from 1 to 5”). Only single-task optimization with MAE validation meaningfully improves (+0.031 Spearman). Without a validation gate, multi-task modes actively degrade: SSC drops from 0.283 to 0.184 by step 7. With the gate, trajectories flatten rather than climb — the gate prevents collapse but cannot manufacture signal from diluted gradients.

Per-task Spearman ρ over 12 optimization steps (mean of 3 runs, shaded bands show min–max). Top row applies MAE validation gating; bottom row has none. Performance degrades as coupling increases: Single > SSS > SSC > SCC > CCC.

Failure Mode 2: Instruction Interference

Gradient dilution explains why the cross-task modes fail. But why do the per-task modes (SSS, SSC) also stagnate, when their gradients are sharp and their edits faithful? The answer lives at inference time, not optimization time.

We run an oracle experiment: for each criterion, we pick the single best instruction across all single-task runs — the one with the highest held-out Spearman for that task — then combine the four oracle-optimal instructions into one prompt. Even these individually-best instructions degrade when combined, falling from 0.305 to 0.220 average Spearman (−0.085), strictly worse than the generic baseline.

The mechanism is instruction-length asymmetry. Optimization over-specifies some criteria (the fluency rubric balloons to ~800 tokens) while leaving others terse (relevance stays at ~4 tokens). Packed into one prompt, the verbose instructions hog the judge's attention budget at inference, drowning out the brief ones. Individually good rubrics can hurt when combined — so interference cannot be fixed by better per-task optimization alone.

DeepSeek v4: Same Failure Modes, Stronger Backbone

We replicate with DeepSeek v4 (Flash as task LLM, Pro as loss/gradient/optimizer). The stronger model yields better absolute performance (initial ~0.38, best up to 0.43), and all modes improve from initialization. But the structural pattern holds: single-task achieves the largest gain (+0.117), while multi-task modes improve modestly (+0.026 to +0.039).

The gradient dilution cliff also persists: per-task modes score 8.8 specificity vs. 4.25 for joint-gradient modes — a 52% reduction. A better LLM lifts all boats but does not close the gap between single-objective and multi-objective optimization.
